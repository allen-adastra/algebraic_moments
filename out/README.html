AlgebraicMoments

AlgebraicMoments is a package for automatically generating code to propagate statistical moments through polynomial expressions (constraints or dynamics).

Background

Statisticial moments are a useful way of characterizing distributions of random variables. Motivated by the problem of chance-constrained motion planning for potentially non-Gaussian systems with nonlinear constraints and dynamics, recent works leverage statistical moments to establish upper bounds on chance-constraints. Doing so, however, often requires the user to either propagate statistical moments through dynamics or compute moments of polynomials applied to random vectors. It turns out that, when working with polynomial dynamics and constraints, it is possible to algorithmically derive closed form expressions for the necessary moments.

Moment Expressions

Letting w\mathbf{w}w denote a random vector, y\mathbf{y}y denote deterministic variables, and ggg denote a polynomial function. The moment expressions capability allows the user to express:

E[g(y,w)n]\mathbb{E}[g(\mathbf{y}, \mathbf{w})^n] E[g(y,w)​n​​]

in terms of y\mathbf{y}y and moments of w\mathbf{w}w in closed form. In the case that ggg is a vector-valued function, AlgebraicMoments can also handle moments of the following form where ni∈Nn_i\in\mathbb{N}n​i​​∈N:

E[∏i=1ng(y,w)ni]\mathbb{E}\left[\prod_{i=1}^n g(\mathbf{y}, \mathbf{w})^{n_i}\right] E[​i=1​∏​n​​g(y,w)​n​i​​​​]

To simplify notation, we express moments of the above form with multi-index notation. Letting α∈Nn\alpha\in\mathbb{N}^nα∈N​n​​ denote a multi-index and αi\alpha_iα​i​​ denote the ithi_{th}i​th​​ element of α\alphaα, multi-index notation is defined as:

E[g(y,w)α]:=E[∏i=1ng(y,w)αi]\mathbb{E}[g(\mathbf{y}, \mathbf{w})^\alpha] := \mathbb{E}\left[\prod_{i=1}^n g(\mathbf{y}, \mathbf{w})^{\alpha_i}\right] E[g(y,w)​α​​]:=E[​i=1​∏​n​​g(y,w)​α​i​​​​]

Using AlgebraicMoments to Derive Moment Expressions

Suppose x=[x,y]\mathbf{x} = [x, y]x=[x,y] is a random vector with x,yx, yx,y dependent and ccc is some deterministic variable. Now suppose we are interested in expressing the following moments in terms of moments of x,yx, yx,y:

g_1 = \mathbb{E}[(cxy^2 + y)^2]\\ g_2 = \mathbb{E}[(x^2y + cy^2)^3]

We can express this problem using AlgebraicMoments as simply as:

# Specify elements of the random vector and dependencies.
x = RandomVariable("x") 
y = RandomVariable("y")
dependencies = [(x, y)] # Specify pairwise dependencies between variables
random_vector = RandomVector([x, y], dependencies)

# Specify the deterministic variables.
c = DeterministicVariable("c")
deterministic_variables = [c]

# Specify the moments we care about as a dictionary, with the keys being the name we want to give the moment and the values being the expression in terms of our random and deterministic variables.
expressions = {"g1" : (c * x*y**2 + y)**2, "g2" : (y*x**2 + c*y**2)**3}

And then generating code for our moment expressions is as simple as:

# Generate an instance of "MomentExpressions" and print
moment_expressions = generate_moment_expressions(expressions, random_vector, deterministic_variables)
moment_expressions.print("matlab")

Concentration Inequalities

For an uncertain ϵ\epsilonϵ chance-constraint expressed as:

P(g(y,w)≤0)≤ϵ\mathbb{P}(g(\mathbf{y}, \mathbf{w})\leq 0)\leq \epsilon P(g(y,w)≤0)≤ϵ

Given the mean and variance of g(y,w)g(\mathbf{y}, \mathbf{w})g(y,w), we can upper bound the above probability using concentration inequalities. This package provides the ability to generate code for the following concentration inequalities:

  • Cantelli
  • Vysochanskij-Petunin (VP)
  • Gauss

Going back to the example for MomentExpressions, suppose we are interested in bounding:

P((cxy2+y)2≤0)\mathbb{P}((cxy^2 + y)^2\leq 0) P((cxy​2​​+y)​2​​≤0)

We can generate the corresponding code with:

concentration_inequality = generate_concentration_inequality((c * x*y**2 + y)**2, random_vec, deterministic_vars, "cantelli")
concentration_inequality.print("python")

Dynamical Systems (WORK IN PROGRESS)

Let xt\mathbf{x}_tx​t​​ denote the state, ut\mathbf{u}_tu​t​​ denote the control, and wt\mathbf{w}_tw​t​​ denote a random vector that is independent of xt\mathbf{x}_tx​t​​. Suppose we have a polynomial system:

xt+1=f(xt,ut,wt)\mathbf{x}_{t+1} = f(\mathbf{x}_t, \mathbf{u}_t, \mathbf{w}_t) x​t+1​​=f(x​t​​,u​t​​,w​t​​)

In many cases, we want to be able to determine statistical moments of xt\mathbf{x}_tx​t​​ for some finite horizion (perhaps we want the mean vector and covariance matrix). In a certain sense, we are trying to determine the dynamics of a moment state. In this package, we introduce two methods for doing so 1) moment shooting and 2) moment state dynamical systems.

Moment Shooting

Moment shooting is analogous to direct shooting in trajectory optimization, we essentially express the statistical moments of xt\mathbf{x}_tx​t​​ in close form in terms of moments of the initial state distribution x0\mathbf{x}_0x​0​​, the control inputs ut\mathbf{u}_tu​t​​, and moments of the disturbance vector wt\mathbf{w}_tw​t​​.

Moment State Dynamical System (MSDS)

The method of moment state dynamical systems (MSDS) involves finding a closed form system of equations that expresses the dynamics of a set of moments we care about. Instead of expressing the dynamics in terms of the state vector xt\mathbf{x}_tx​t​​, a MSDS expresses the dynamics of a set of moments of xt\mathbf{x}_tx​t​​. With this package, the user can specify a system and a set of moments they care about. Before going any further, we need to introduce some notation. Recall multi-index notation from the section on Moment Expressions; note how there is a one-to-one mapping between statistical moments of xt\mathbf{x}_tx​t​​ and multi-indices in Nn\mathbb{N}^nN​n​​. Thus, we can represent moments with multi-indices, and we will call sets of multi-indices "moment bases" and the corresponding set of moments "moment states". Letting A\mathcal{A}A denote a moment basis, the moment state will be denoted by:

xt[A]:={E[xtα]:α∈A}\mathbf{x}_t[\mathcal{A}]:=\{\mathbb{E}[\mathbf{x}_t^\alpha] : \alpha\in\mathcal{A}\} x​t​​[A]:={E[x​t​α​​]:α∈A}

As a simple example, consider x=[x,y]\mathbf{x} = [x, y]x=[x,y] with the moment basis A={(1,0),(0,1),(2,0)}\mathcal{A} = \{(1,0), (0, 1), (2, 0)\}A={(1,0),(0,1),(2,0)}. The moment state would then be:

x[A]={x,y,x2}\mathbf{x}[\mathcal{A}] = \{x, y, x^2\} x[A]={x,y,x​2​​}

The idea is that the user can specify a dynamical system and a moment basis (or state) that they care about and then an algorithm called TreeRing (named as such because it is essentially tree search over the ring of polynomials) will search for a higher dimensional moment basis A\mathcal{A}A and the corresponding set of dynamics equations hhh such that:

xt+1[A]=h(xt[A],wt[B],ut)\mathbf{x}_{t+1}[\mathcal{A}] = h(\mathbf{x}_t[\mathcal{A}],\mathbf{w}_t[\mathcal{B}], \mathbf{u}_t) x​t+1​​[A]=h(x​t​​[A],w​t​​[B],u​t​​)

Above B\mathcal{B}B is a moment basis for the disturbance vector and wt[B]\mathbf{w}_t[\mathcal{B}]w​t​​[B] denotes a set of moments of the disturbance vector. This is a powerful idea because we can simulate the statistical moments of state in the future in a manner that scales linearly w.r.t. the number of time steps.
